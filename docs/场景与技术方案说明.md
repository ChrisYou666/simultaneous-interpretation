# 场景与技术方案说明

本文说明本项目的**目标业务场景**、**推荐技术路线**，以及**与当前代码实现的关系**，便于产品、研发与集成方对齐。

---

## 一、目标业务场景

### 1.1 场景定位

- **会议环境下的同声传译（SI）**：面向现场或线上会议，听众需要**连续听到**译文，而非仅会后阅读文本。
- **语种范围：中文（zh）、英语（en）、印尼语（id）**，三种语言之间需要互译能力。
- **一路发言 → 两路收听**：发言人使用**一种语言**发言时，系统需同时产出**另外两种语言**的语音，并通过**不同声道/不同音频轨**分发给听众（例如：听众 A 只听英文轨，听众 B 只听印尼文轨，或立体声分路）。
- **会中语言不唯一**：同一场会议中可能出现**多种语言交替汇报**（中/英/印切换，或同一段中夹杂多语），系统需能识别或配置**当前发言语言（Floor language）**，再驱动正确的识别与翻译方向。

### 1.2 非目标（当前阶段可明确排除或降级）

- 不等同于「仅大模型对话框翻译」或「上传文档翻译」——后者可作为**辅助**，但不是会场连续同传的主路径。
- 若未单独建设**流式语音链路**，则现有能力定位为**备稿试译、术语与材料增强、会后稿**，而非完整「同声传译产品」。

---

## 二、推荐技术方案（目标架构）

会场级中英印三语、双路语音输出，建议采用**低延迟流式管线**，与「整段文本调用大模型」解耦。

### 2.1 逻辑链路（自上而下）

```
会场拾音 / 发言人麦克风
        ↓
流式语音识别（ASR），支持当前源语配置或语言识别（LID）
        ↓
稳定文本片段（按标点、时长或语义切段）
        ↓
机器翻译（MT）×2：源语 → 目标语 A、源语 → 目标语 B
        （三语中排除当前源语后的两种目标语）
        ↓
语音合成（TTS）×2：并行生成两路目标语音
        ↓
双路音频输出：Web/App 双 AudioTrack、或接入调音台/同传接收机不同频道
        ↓
听众端：选择「听英文」「听印尼文」或分路立体声等策略
```

### 2.2 关键设计要点

| 要点 | 说明 |
|------|------|
| **Floor 语言** | 可由**会议控台**手动切换「当前发言语种」，或与 **LID** 结合；混讲场景建议**人工兜底**。 |
| **三语互译** | 工程上可采用**枢纽语**（如经英语中转）或**多语翻译引擎**；需权衡延迟与质量。 |
| **双路对齐** | 两路 TTS 独立缓冲，需处理**延迟差**与**听感连续**，避免两路严重不同步。 |
| **延迟预算** | ASR、MT、TTS 串联，需整体评估端到端延迟；流式 ASR、分段 MT、流式/低延迟 TTS 为常见组合。 |
| **大模型角色** | 适合：**术语表、人名与机构、会议材料（PDF）与图片**的增强、**试译与会后润色**；不宜单独作为唯一实时同传引擎（除非严格做短句级与强 SLA）。 |

### 2.3 分发形态

- **纯软件**：浏览器/WebRTC 两路音频轨；听众用耳机在手机或网页选语言轨。
- **线下会场**：数字音频矩阵或同传系统，将两路 AI 输出映射到**红外/射频不同频道**（与传统同传听众机兼容思路类似，声源由 AI 替代人工同传）。

## 三、与当前仓库实现的关系

### 3.1 已实现能力（摘要）

- **前端**：会议式 UI（时间轴、双栏、麦克风等布局）、**模型试译**、**关键词/上下文/会议材料（含 PDF）与图片**注入翻译请求。
- **后端**：Spring Boot，**OpenAI 兼容 Chat API**，JWT 与管理员配置（API、主模型），`si_setting` / `si_model_app` 等配置落库。

上述能力适用于：**材料辅助、段落试译、多模态提示**，为同传场景提供**内容增强**与**管理配置**基础。

### 3.2 与目标场景的差距

| 目标场景需求 | 当前实现侧重 |
|--------------|----------------|
| 连续语音进、连续语音出 | 以**文本片段 + 可选图片**调用模型为主 |
| 双路 TTS、双声道分发 | 未包含 **TTS 与多轨音频调度** |
| 流式 ASR、会中切语言 | 未包含 **流式 ASR / Floor 控台** |
| 端到端延迟优化 | 未按**毫秒级同传链路**优化 |

### 3.3 建议演进路径

1. **阶段 A（POC）**：一路拾音 → 流式 ASR → 固定 Floor 语 → 双路云 TTS → 浏览器双轨播放，验证延迟与稳定性。  
2. **阶段 B（三语产品化）**：Floor 切换、中英印方向配置、术语表与现有「材料增强」对接。  
3. **阶段 C**：可选 LID、控台、与硬件/会议平台集成。
